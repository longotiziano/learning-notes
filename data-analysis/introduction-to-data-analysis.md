# Introducción al análisis de datos

## Tipos de análisis de datos
1. **Análisis descriptivos**: Responde a la pregunta del qué pasó, y provee información acerca
de eventos pasados
2. **Análisis diagnóstico**: Toma la información proveniente del análisis descriptivo y lo 
investiga de manera más profunda, buscando la causa de lo ocurrido.
3. **Análisis predictivo**: Muestra lo que podría llegar a ocurrir.
4. **Análisis prescriptivo**: Analiza decisiones pasadas y eventos para ver qué acción tomar.

## Proceso del análisis de datos
1. Entender el problema y el resultado al que querés llegar.
2. Colocar una métrica clara: ¿qué voy a mostrar y cómo lo voy a medir?
3. Recién ahí te fijás fuentes de datos y herramientas requeridas.
4. Limpieza de datos. Este es un proceso crítico, ya que la calidad de los datos depende de los 
resultados.
5. Analizar los datos desde diferentes perspectivas, entender tendencias, identificar 
correlaciones, patrones, variaciones...
6. Interpretar resultados: si en análisis se defiende frente a todos los contraejemplos posibles.
7. Presentar resultados de manera clara, convincente, impactante y gráfica.

## Responsabilidades del analista de datos
- Adquirir datos, crear consultas, filtrar, limpiar, estandarizar, reorganizar, uso
de herramientas y técnicas estadísticas, analizar patrones, preparar reportes y gráficos y
finalmente documentar el proceso.
- Combinación entre habilidades:
    - **Técnicas**: En spreadsheets, software de visualización, programación, SQL, extracción
    de datos proveniente de repositorios, familiaridad con el Big Data...
    - **Funcionales**: Estadística, analítica, solución de problemas, correcta identificación
    de problemas y resultados, creación que gráficos claros, manejo de proyectos...
    - **Soft skills**: Trabajar en equipo, comunicación efectiva, story-telling, curiosidad,
    intuición...

## Día de un analista de datos
- Al momento de resolver una problemática, es importante y útil plantear las hipótesis iniciales.
- Siempre documentar el proceso para poder mostrarlo de manera efectiva y convincente.

## Identificación de datos para análisis
1. Determinar la información que querés conseguir
    - Información que necesitás
    - Posibles fuentes de información para esos datos
2. Definir el plan de obtención
    - Timeline para conseguir la información
    - Cuánto es suficiente
    - Dependencias, riesgos y plan de mitigación
3. Determinar tus métodos
    - Fuentes de datos
    - Tipos de datos
    - Tiempo requerido
    - Volumen
- Luego de realizar todo el plan, puedes comenzar. Vas a realizar actualizaciones porque las
condiciones evolucionan a medida que avanzás.
- Los datos identificados tienen implicaciones de:
    1. **Calidad**: Libre de errores, precisa, completa, relevante, accesible...
    2. **Seguridad**: Regulación, compliances... 
    3. **Privacidad**: Confidencialidad, licencia de uso, regulaciones... Se tienen que definir
    checks, validaciones y el flujo de auditoría.
> Esto que el curso está contando es muy interesante! Ojalá haberlo sabido antes de comenzar 
proyectos

## Fuentes de datos
- **Primary data**: Datos obtenidos de manera directa de la fuente, como por ejemplo entrevistas,
discusiones, observaciones... TAL CUAL SALIÓ DEL ORIGEN, DEL PROVEEDOR.
- **Secondary data**: Información obtenida desde fuentes ya existentes, como bases de datos,
artículos, publicaciones, búsquedas de internet... También aplican los ejemplos de las
fuentes primarias en caso de que estén conducidas de manera externa. DATOS ANALIZADOS,
INTERPRETADOS O PROCESADOS, el autor NO genera los datos originales.
- **Third-party data**: Datos comprados de desarrolladores que recojen datos de distintas 
fuentes y los combinan en datasets con el propósito de venderlos.
